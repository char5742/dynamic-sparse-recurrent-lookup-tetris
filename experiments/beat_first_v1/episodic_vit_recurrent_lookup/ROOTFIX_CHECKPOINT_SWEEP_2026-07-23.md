# 根本修正版EVRLの10,000更新刻みcheckpoint再評価

## 結論

追加学習を行わず、根本修正版EVRLの10,000～100,000更新checkpointを同一パネルで
再評価した。主採用checkpointは100,000更新ではなく、**90,000更新**である。

90,000更新はcomposite lossが全checkpoint中で最小であり、top-1は最高値に同率、
pairwiseは単独最高だった。NDCGも単独最高の70,000更新との差が`0.000113`に留まる。
一方、100,000更新はaction marginだけは最高だったが、loss、NDCG、pairwiseが90,000
更新から反落し、平均深度も`3.027`から`4.880`へ増えた。この固定パネルでは、
90,000更新以降の追加計算が総合品質の改善につながっていない。

## 学習推移

### 10,000～30,000更新：基礎順位学習

最も大きく改善した区間である。lossは`2.778579 -> 2.584743`、top-1は
`0.523438 -> 0.640625`、NDCGは`0.981363 -> 0.988974`、pairwiseは
`0.854635 -> 0.895458`へ改善した。平均深度はほぼ2のままなので、この初期改善は
再帰回数の増加ではなく、入力表現、長期記憶、順位headの基礎学習によるものと読める。

Lookup側では平均row-load Giniが`0.426669 -> 0.282886`へ低下し、平均coverageが
`85.15% -> 99.21%`へ上昇した。anti-collapse学習は最初の30,000更新でほぼ全rowへ
探索範囲を広げている。

### 30,000～60,000更新：緩やかな改善と停滞

lossは`2.584743 -> 2.563182`、NDCGは`0.988974 -> 0.990876`、pairwiseは
`0.895458 -> 0.906285`へ緩やかに改善した。一方、top-1は`0.640625 -> 0.632812`
で横ばい圏にあり、50,000更新と60,000更新のlossもほぼ同じだった。

平均深度は`2.038 -> 2.820`へ増えたが、top-1の改善には直結しなかった。この区間では
Lookupの均衡化は続いているものの、離散的な正解候補を選ぶ能力は明確な踊り場に入った。

### 60,000～70,000更新：再帰利用を伴う第2の改善

70,000更新でloss`2.555018`、top-1`0.664062`、NDCG`0.992014`、
pairwise`0.913456`へ同時に改善した。平均深度も`2.820 -> 4.273`へ増えた。
初期区間と異なり、この改善は再帰深度の増加と同時に起きている。

ただし、これだけでは「深く考えるほど常に良い」とは言えない。次の80,000更新で平均深度が
最小値2へ戻り、品質も一部反落したためである。

### 70,000～90,000更新：haltingの振動を伴う最良点への到達

80,000更新では平均深度が`2.000`へcollapseし、top-1が`0.625000`へ反落した。
それでもpairwiseは`0.913632`で維持された。90,000更新では平均深度が`3.027`へ戻り、
loss`2.551617`、top-1`0.671875`、pairwise`0.913689`で総合最良となった。

従って、70,000～90,000更新は単純な右肩上がりではない。連続順位指標は高い水準を保つ
一方、hard haltingの方策変化によってtop-1と平均深度が大きく振動している。

### 90,000～100,000更新：margin拡大と総合品質の反落

100,000更新ではaction marginが`0.116655 -> 0.137023`へ拡大し、平均深度も
`3.027 -> 4.880`へ増えた。しかしlossは`2.551617 -> 2.559976`、NDCGは
`0.991901 -> 0.991213`、pairwiseは`0.913689 -> 0.908830`へ悪化した。

これは、より深い再帰と大きいscore marginが、同一パネル上の順位全体の改善を保証しない
ことを示す。90,000更新以降は「未収束でまだ一様に伸びる区間」ではなく、複数目的と
halting方策が振動する頭打ち域と判定する。

| 指標 | checkpoint | 値 |

単一指標ではcheckpointが分かれる。90,000更新は、最小loss、最高top-1同率、最高
pairwiseを同時に満たし、NDCGも最高値から128状態パネル上で実質的に近い。そのため、
特定の一指標だけを最大化せず総合順位品質を選ぶcheckpointとして最も妥当である。

## 100,000更新との差

90,000更新を100,000更新と比べると、次の差になった。

| 指標 | 90,000 - 100,000 | 判定 |
|---|---:|---|
| composite loss | -0.008359 | 90,000が良い |
| top-1 | 0 | 同率 |
| NDCG | +0.000688 | 90,000が良い |
| pairwise | +0.004859 | 90,000が良い |
| action margin | -0.020369 | 100,000が良い |

100,000更新への延長はmarginを広げた一方、主要な順位指標を改善しなかった。従って、
「100,000更新完走checkpoint」と「同一パネル上の採用checkpoint」を区別して保存する。

## routingと再帰の推移

Lookup row-load Giniは10,000更新の`0.418～0.440`から、90,000更新の
`0.193～0.196`まで継続的に低下した。coverageも`83.9～87.2%`から
`99.7～99.9%`へ上がった。従って90,000～100,000更新の品質反落をLookup collapseの
再発とは解釈しない。

平均深度は70,000更新で`4.273`、80,000更新で`2.000`、90,000更新で`3.027`、
100,000更新で`4.880`と揺れている。haltingは入力依存の可変深度を維持しているが、
学習後半の深度増加は単調な品質向上を意味しない。今後checkpointを選ぶ場合も、
最終更新数や平均深度ではなく固定パネルの順位品質で決める。
